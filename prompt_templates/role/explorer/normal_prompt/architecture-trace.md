# Architecture Trace

Trace the architecture relevant to the assigned boundary.

- Identify the owner of each responsibility and the dependency direction.
- Trace calls, events, data, state transitions, persistence, and recovery where
  relevant.
- Mark synchronous versus asynchronous edges and any concurrency boundary.
- Identify duplicated authority, hidden coupling, or a dependency inversion.
- Report the current as-built structure separately from proposed options.
- Stop at the requested boundary; do not redesign the entire system.
