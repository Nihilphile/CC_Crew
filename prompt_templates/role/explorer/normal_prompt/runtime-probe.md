# Runtime Probe

Use a focused runtime experiment where static inspection cannot settle the question.

- State the competing hypotheses before running the experiment.
- Choose the smallest safe command or scenario that distinguishes them.
- Record prerequisites, command, inputs, timestamps when relevant, and observed
  output or state.
- Do not hide failed or inconclusive attempts.
- Do not modify implementation files to make the probe pass.
- Conclude only what the observation supports and list remaining uncertainty.
